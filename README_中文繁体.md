# 智能災備切換平台 (Smart Disaster Recovery Manage System)

> **業務連續性保障的最後一道防線 —— 自動化、可視化、智慧化的災備營運管理平台**

## 📖 專案簡介

**智慧災備切換平台** 是一款面向企業級資料中心與雲端環境的災備營運管理產品。它透過 **流程可視化編排、自動化執行、即時監控與 AI 輔助決策**，徹底解決了傳統災備切換中流程複雜、依賴人工、驗證困難、RTO 過長等核心痛點。

本平台嚴格遵循國際與國內災備及業務連續性管理標準（SHARE78、ISO 22301、GB/T 20988），為企業提供從災備建設、預案編制、應急演練到實際切換的全生命週期管理能力，確保核心業務系統在災難發生時能夠 **分鐘級恢復**（RTO ≤ 30 分鐘，RPO ≤ 5 分鐘）。

## ✨ 核心功能

### 1. 容災切換流程可視化編排 (BPMN 2.0)
- **圖形化拖曳設計**：基於標準 BPMN 2.0 規範，透過拖曳方式靈活編排單一業務或複雜跨系統切換流程。
- **多場景適配**：支援桌面演練（邏輯推演）、模擬演練（鏈路驗證無感資料）及實際切換。
- **參數化與腳本管理**：內建自動化作業節點與參數配置，無縫銜接底層維運腳本。

### 2. 全自動化／半自動化切換執行
- **跨平台支援**：相容 Linux/Unix 衍生系統、Windows、PowerVM-AIX、VMware 虛擬化。
- **廣泛的應用覆蓋**：支援 WebLogic、WebSphere、Nginx、IBM MQ、Oracle、MySQL、DB2、SQL Server 等主流中介軟體與資料庫的自動啟停與狀態檢測。
- **高併發執行**：透過自動化引擎大幅提升併發度，將恢復時效從小時級壓縮至分鐘級。

### 3. 可觀測性與可視化驗證
- **切換過程即時大螢幕**：對切換進度、資源狀態、資料複製鏈路進行全鏈路監控。
- **自動化驗證**：切換完成後自動執行應用程式存活探測與資料一致性校驗，確保業務可用性。

### 4. AI 輔助災備營運 (智慧維運 AIOps)
- **風險主動防禦**：基於歷史資料與即時指標預測潛在風險，變被動回應為主動規避。
- **智慧資源調度**：提供成本最佳化建議，提升災備資源 ROI。
- **合規報告自動生成**：一鍵生成滿足監管要求的審計報告，全程操作留痕可追溯。

### 5. 靈活的審批與流程控制
- **自訂審批流程**：支援按業務系統配置審批節點、會簽／或簽邏輯。
- **執行期干預**：支援人工互動確認、暫停、跳過、一鍵退回，確保切換過程絕對可控。

## 📄 文件與使用場景

- **場景一：日常容災演練**  
  使用「模擬演練」模式，在不影響生產業務的情況下驗證災備環境連通性與腳本正確性。

- **場景二：計畫內停機切換**  
  透過 BPMN 編排「停用應用程式 → 確認資料同步 → IP 漂移 → 災備端啟動應用程式」的標準流程，一鍵執行。

- **場景三：突發故障應急回應**  
  藉助 AI 預測風險，提前預熱災備資源，支援決策者一鍵發起緊急切換流程。
